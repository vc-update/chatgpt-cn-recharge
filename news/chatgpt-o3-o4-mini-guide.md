## 一、o3 和 o4-mini 是什么？

### 1.1 推理模型 vs 对话模型

ChatGPT 的模型可以分为两类：

| 类型 | 代表模型 | 核心特点 |
|:---|:---|:---|
| 对话模型 | GPT-4o、GPT-5.4 | 响应快、流式输出、适合日常对话 |
| **推理模型** | **o3、o4-mini** | **回答前"深度思考"，适合复杂问题** |

推理模型在回答之前会进行一段内部"思考"过程（类似人类打草稿），因此在**数学、逻辑、代码调试、复杂分析**等任务上远超普通对话模型。

### 1.2 o3 和 o4-mini 的定位

- **o3**：OpenAI 旗舰推理模型，能力最强，适合最复杂的任务，每月使用有配额限制
- **o4-mini**：轻量推理模型，速度更快、配额更高，适合日常推理需求，性价比极高

::: tip 一句话记住区别
o3 = 最强大脑，用于攻克难题；o4-mini = 高效大脑，用于日常推理
:::

## 四、如何切换到 o3 或 o4-mini？

### 4.1 网页版切换方法

1. 打开 [chatgpt.com](https://chatgpt.com)（需科学上网）
2. 在对话界面顶部找到**模型选择下拉菜单**
3. 点击当前模型名称（如 GPT-5.4）
4. 从列表中选择 **o3** 或 **o4-mini**
5. 开始新对话即可使用所选模型

::: tip 小技巧
切换模型后，这个设置会保留到下次你手动更换。如果只是偶尔需要推理模型，记得任务完成后切回 GPT-5.4，节省 o3 配额。
:::

### 4.2 手机 App 切换方法

1. 打开 ChatGPT App（iOS 或 Android）
2. 点击对话界面顶部的模型名称
3. 在弹出列表中选择 o3 或 o4-mini
4. 返回对话界面开始使用

### 4.3 看到"Thinking..."是正常的

选择 o3 后发送消息，你会看到 **"Thinking..."** 的等待提示，这是推理过程，不是卡住了。复杂问题可能要等 30 秒到几分钟，请耐心等待。

## 五、o3 和 o4-mini 最适合做什么？

### 5.1 最适合 o3 的场景

**竞赛级数学题**
```
例：证明√2是无理数，并用至少3种不同方法求解
```
o3 会逐步展示推理过程，像顶尖数学家一样验证每个步骤。

**复杂代码调试**
```
例：帮我找出这段 Python 代码中的内存泄漏，并给出修复方案：[粘贴代码]
```
o3 能追踪多层调用栈，定位普通模型找不到的隐藏 bug。

**多步骤逻辑推理**
```
例：分析这份商业计划书的逻辑漏洞，给出改进建议
```

**法律/医疗/金融等专业分析**
```
例：分析这份合同条款是否存在对甲方不利的条款
```

### 5.2 最适合 o4-mini 的场景

- 日常编程辅助（代码生成、review）
- 数据分析和统计计算
- 学习辅导（解题步骤展示）
- 逻辑题和谜题求解
- 性价比高的推理任务

### 5.3 不适合用推理模型的场景

以下任务用 GPT-5.4 更合适，不要浪费 o3 配额：
- 日常聊天和问答
- 写作和文案创作
- 简单的信息检索
- 图片生成（推理模型不生成图片）

## 六、o3 使用配额管理技巧

Plus 用户的 o3 配额是有限的（约 50 次/天），以下技巧帮你高效利用：

**1. 简单问题用 o4-mini，复杂问题才用 o3**

o4-mini 的配额远高于 o3，处理大多数推理任务完全够用。只有遇到 o4-mini 也解决不了的超难问题，才切换 o3。

**2. 一次提问包含所有信息**

由于 o3 每次使用都消耗配额，确保第一次提问时把所有相关信息都给到，避免因为补充问题而多次消耗配额。

**3. 记录剩余配额**

在模型选择界面旁边，ChatGPT 会显示当前的配额剩余情况，注意观察。

## 七、常见问题

### Q：为什么我的 Plus 账号看不到 o3？

可能原因：
- 账号未升级为最新的 Plus 计划（老用户需要重新确认订阅状态）
- 使用了不支持 o3 的地区
- 浏览器缓存问题（清除缓存后重试）

### Q：o3 和 o3-mini 是一样的吗？

不一样。o3-mini 是 o3 的早期轻量版，已被 o4-mini 取代。目前主推 o3（旗舰推理）和 o4-mini（轻量推理）。

### Q：使用 o3 会额外收费吗？

Plus 订阅费用已包含 o3 的基础使用额度，不会额外收费。但如果超出每日配额，需要等待次日重置或升级 Pro。

### Q：o3 生成的内容准确吗？

o3 在事实准确性上优于大多数模型，但仍然可能出错。对于重要决策，请自行验证关键信息。
